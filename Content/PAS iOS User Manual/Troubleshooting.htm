Troubleshooting

If you receive repeated error messages or the InTouch Provider Access Software fails to respond, completely close the App and start it again. See “Shutting Down the InTouch Provider Access Software for iOS on page 1”.

Login Problems

The following are messages that may occur when you try to login to the InTouch Telehealth Network using the InTouch Provider Access Software. These problems are specific to the iPad / iPhone itself or the network that the iPad / iPhone is connected to, not the Patient Access Device.

Turn Off Airplane Mode

Cause:

  • The iPad / iPhone does not have network access because it is in Airplane Mode.

Solution:

  1. Go to the Settings page and turn off Airplane Mode.
  2. Ensure you have a Wi-Fi or Cellular network connection.

Not Connected to Internet

Cause:

  • The iPad / iPhone cannot access the Internet. If you cannot access the public Internet then you will not be able to access the InTouch Telehealth Network or any Patient Access Devices.

Solution:

  1. Ensure you have a Wi-Fi or Cellular network connection. It is not enough to just be on a network; the network must have Internet access.
  2. Bring up Safari or other browser on the iPad / iPhone and verify that you can access a common web site, such as www.google.com.
  3. If you are on a public or hotel Wi-Fi network you may have to enter a guest username/password and accept their terms & conditions.
  4. Try resetting the network connection by turning Airplane mode on and off again.

NOTE: This is an Internet access problem, not a username/password problem.

Login Failed

Cause:

The InTouch Telehealth Network did not accept the Username/Password combination. There is no way to know whether the problem is with the Username or with the password.

Solution:

Make sure your Username and Password are correct.

  • The password is case sensitive.
  • Your Username is not case sensitive; it doesn’t matter if you use upper or lower case letters.
  • In rare cases, completely shutting down the InTouch Provider Access Software may correct the problem. See “Shutting Down the InTouch Provider Access Software for iOS on page 1”.
  • If you enter an invalid password five times in a row, the server will lock your account. Wait 60 seconds to try again or reset your password by tapping the Forgot Password link on the login page.

NOTE: This is not a firewall problem.

Server Unreachable

Cause:

The iPad is connected to the Internet but did not receive a response from the InTouch Telehealth Network. The most common causes are:

  • A change in the iPad network connection.
  • A network firewall is blocking TCP port 5061.
  • Poor network quality

Solution:

Sometimes the InTouch Provider Access Software does not cleanly handle going out of and into Wi-Fi and/or cellular coverage. The first and easiest thing to try is to completely shut down the app, re-launch it, and then try to login again. See “Shutting Down the InTouch Provider Access Software for iOS on page 1”.

  • If this is a firewall problem and you are on a guest network, such as at a coffee shop or hotel, then you will not be able to change the firewall settings. Your only option is to get on another network. Generally speaking, cellular connections do not have firewall problems.
  • Try resetting the network connection by turning Airplane mode on then back off again.

NOTE: This is not a username/password problem.

Patient Access Device Connection Problems

The following are messages that may occur after you have successfully connected to the InTouch Telehealth Network, when you try to connect (beam in) to a Patient Access Device.

All the messages will say, Unable to connect to endpoint at the top. It is imperative that you read the text of the message carefully to see the particular type of failure.

Patient Access Device Unavailable

Cause:

The InTouch Telehealth Network tried to contact the Patient Access Device but it did not get a response. It may be due to firewall settings on the hospital (Patient Access Device) network.

Solution:

The Patient Access Device might be turned off or it has lost network access. Contact Technical Support.

 

 

NOTE: This is not an iPad network problem.

Patient Access Device Busy

Cause:

The Patient Access Device is already in a session.

Solution:
There is nothing you can do except try again later. Normally the Patient Access Device will immediately become available when a session ends but in some cases it may take up to a minute.

 

 

Not Authorized to Connect

Cause:

The user is not authorized to connect to the given Patient Access Device. This is an InTouch Telehealth Network connectivity permissions issue.

Solution:

Contact Technical Support.

Network Changed

Cause:
The software has detected that the iPad’s network has changed. This is a normal and expected message when your network connection changes from no Wi-Fi to Wi-Fi, for example, or no network to cellular.

Solution:
The software needs to reset its internal state; login on the new network connection.

Problems in Session

No Audio from InTouch Provider Access Software

  • When in a session, you receive audio from the Patient Access Device but you are not heard at the Patient Access Device.
  • The iOS operating systems have a privacy setting that controls which software programs can access the iPad / iPhone microphone. Open your Settings and select your Privacy settings.
  • Select Microphone from the Privacy Settings menu to check that InTouch Health app has access to the microphone.

No Video from InTouch Provider Access Software

  • When in a session, you receive video from the Patient Access Device but you are not seen on the Patient Access Device.
  • The iOS operating systems have a privacy setting that controls which software programs can access the iPad / iPhone camera. Open your Settings and select your Privacy settings
  • Select Camera from the Privacy Settings menu to check that InTouch Health app has access to the camera.

Network Faults

The InTouch Provider Access Software will display error messages in the lower right of the Remote Camera View when conditions are detected affecting the remote session. Some of these error messages will direct you to the problem, while others will recommend that you contact InTouch Health Technical Support.

Message Explanation Action
“Poor network conditions.” Total interruption in network packets occurred for a significant period of time. Possible cause: poor connectivity, system being powered down or being disconnected from public network. Wait for Internet connection to return, or disconnect.
“Poor network conditions.” Very high round-trip latency at InTouch Provider Access Software or Patient Access Device. Possible cause: poor internal network conditions, low Quality of Service Internet connection or excessive bandwidth usage. These error messages represent non-optimal Internet connection conditions. They may happen periodically on many networks under normal conditions. However, if they persist, contact the hospital’s IT department.
“Poor network conditions.” Packets lost. May see jumpy video with lower frame rates at InTouch Provider Access Software or Patient Access Device. Possible cause: poor internal network conditions, low Quality of Service Internet connection or excessive bandwidth usage.

Status Messages

Status Messages will be displayed in the lower left of the Remote Camera View indicating conditions of the Patient Access Device
