MultiPRESENCE on an iOS Device with

Providers have the option to use 's MultiPRESENCE from their iOS devices to connect to a Patient Access Device.

The status of each device is represented by the icon to the left of the device name and if MultiPRESENCE is available, the three vertical dots, are displayed to the right of the device name. Click the three vertical dots to open the options' menu.

Note: While connected to a Patient Access Device, you can still access other Apps and even make or accept calls on your iOS device by pressing the home button. To return to the Patient Access Software, tap the red bar (see below) at the top of your iOS screen.

Connecting to a Patient Access Device as a Host from an iOS device

  1. Locate the Patient Access Device to connect to from the My Care Locations screen.
  2. Tap the three vertical dots .
  3. Select Connect as Host from the pop-up menu.
  4. Accept or Decline any Guiests who are waiting to joing the session when a Join Request is received.

Note: When connecting as a Host in MultiPRESENCE, you must connect using the three vertical dots instead of tapping the Patient Access Device name. Tapping on the Patient Access Device name will connect to the device without MultiPRESENCE. If this occurs, disconnect from the device and reconnect using the three vertical dots. ()

iOS MultiPRESENCE Host Controls

To change the options for a MultiPRESENCE guest on either an iOS device, tap the guest's image to access the MultiPRESENCE host control.

 

Option Action
  • Allow the InTouch Device to hear and see this user
    		• Allows the host to toggle selected guest's audio and video on the remote device.
  • Make this user the host
    		• Transfers the host role to the selected guest and changes the current host to a guest role.

    Disconnect this user

    		Disconnects the selected guest from the current MultiPRESENCE session.
