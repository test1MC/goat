MultiPRESENCE Host

The Host of the MultiPRESENCE session will see the screen below on their Provider Access Software. The MultiPRESENCE panel on the left side of the screen shows thumbnail images of each Guest.

NOTE: If there are more than two four Guests present, a scroll bar will appear to enable the Host to view all the thumbnail images.

The Host controls the access each Guest is allowed in the session using a right-click menu. Right click on any thumbnail of a Guest and set the session access for that Guest from the following choices:

  • Allow the Patient Access Device to hear and see this user
  • Allow this user to see the Patient Access Device video
  • Make this user the host
  • Remote viewers see this user's face (instead of yours)
  • Disconnect this user

On the Host interface, when the mouse hovers over the Received or Sent meters a tooltip pops up displaying the responsible party for the low bandwidth. See “Optimizing Bandwidth Usage on page 1”.

NOTES:

  • The default access setting for a Guest joining the session as a full participant is that the Guest is able to be seen and heard on the Patient Access Device.
  • To change the default to not allow Guests to be seen and heard on the Patient Access Device, contact you InTouch Health representative for assistance.

 

When the Host selects Allow the RP Device to hear this user, the Host sees an icon on the Guest's thumbnail image depicting a speaker pointing to an Patient Access Device.

Up to three channels of simultaneous audio are allowed: Patient Access Device, Host Provider Access Software, and one Guest Provider Access Software.

If the Push-to-talk mode is enabled, Guests must either (1) click and hold down the Push to Talk button, or (2) hold down the Shift key while clicking the Push to Talk button, releasing the Shift key will disengage Push to Talk, or (3) Press and hold down the Space Bar or release to disengage. Only one Guest can speak at a time.

In Push-to-talk mode, other MultiPRESENCE users are informed of the current speaker with a medium-sized flashing icon overlaid on the speaker's thumbnail picture.

Conferring Mode:

The Host has the option of using the Conferring Mode to enable the Host and Guests to conference privately amongst themselves without the Patient Access Device hearing or seeing this conference. This selection is located on the Host interface.

 

In Conferring Mode all audio and video to the Patient Access a Device is blocked to enable private conference between Host and Guests only. The Patient Access Device displays a screen advising that the Patient Access Device audio and video are still active.

Disconnecting from a Session as Host

Clicking Disconnect opens a dialog box giving the host the option to:

  • Disconnect and end the session for everyone.
  • Disconnect and transfer the host role to a selected guest.
