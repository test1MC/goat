Starting a MultiPRESENCE Session from a Windows Device

In MultiPRESENCE mode, multiple Guest users may be connected to the Patient Access Device. The session is controlled by a Host user.

Once you log on, you will see the My Care Locations list.

My Care Locations

  1. Locate the device to connect to.
    • See Setting Default MultiPRESENCE® Role on page 1 for how to set a default role.
  2. Select the type of connection from the drop down list:
    • Connect as Host - The host has the ability to control access of guest users and how they interact with the remote session. The host also has control of the Patient Access Device.
    • Connect as Guest - There can be multiple guests during a MulitRESENCE session. The guests have limited capability during the session. The host has the ability to transfer the host status to any guest and can also remove a guest from the session.
    • Connect without MultiPRESENCE - allows for only one user to log into a session. Will show the device as unavailable for other users.

Notes

  • If a Guest is the ﬁrst to connect, they are requested to stand by for the Host to start the MultiPRESENCE session.
  • If the Host has already connected, the Patient Access Device status shows the MultiPRESENCE icon, , in the My Care Locations list.
  • When connecting as a Host in MultiPRESENCE, you must connect using the Host option in the connection drop-down instead of clicking Connect next to the Patient Access Device name. Clicking Connect will connect to the device without MultiPRESENCE. If this occurs, disconnect from the device and reconnect to the device by selecting Host from the Connect drop-down menu.

After Connecting to a Patient Access Device

  1. After connecting, the Guest will see the dialog box below. Guest clicks OK to join the session.
  1. The Host selects to Accept the Guest, provide Audio Only, or Deny the Guest.

 

 

If the Host accepts the Guest into the session, the following popup message appears recommending the use of headphones. If the Host has also selected to force Push-to-talk in their Audio/Video options, the Guest will be notified at the same time. Refer to Forced Push-to-Talk on page 1.

After accepting the Guest, the The Host will see a thumbnail video of the Guest in the MultiPRESENCE Panel.

Accept or deny any guests who have already logged into the remote session when prompted:

  1. If additional Guests request to join, the Host gets a pop up. Host can accept additional Guests in the same way as for the first Guest.
  2. If there is more than one Guest or the Host has selected to force Push-to-talk in their Audio/Video options, all Guests will now get a pop up reminder that they will need to use the Push to Talk button.

In this mode, Guests must either (1) click and hold down the Push to Talk button, or (2) hold down the Shift key while clicking the Push to Talk button, releasing the Shift key will disengage Push to Talk, or (3) Press and hold down the Space Bar or release to disengage. Guest clicks OK to confirm this before continuing.
